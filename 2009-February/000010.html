﻿Bonjour,

Tout d'abord, merci pour votre intérêt pour Cooperation-iws.

* Pour l'écran noir au premier boot: c'est normal, il faut attendre que 
le système copie les données sur la partition persistente. Cela n'arrive 
qu'une fois au premier démarrage et cela peut être assez long suivant la 
vitesse d'écriture de votre clef USB. Il ne faut surtout pas éteindre 
votre machine avant.

* Pour la mise à jour du système d'explotation: cela n'est pas possible 
car seule les applications web sont en mode lecture, écriture. Installer 
de nouvelles applications n'est pas possible non plus (sauf pour les 
applications webs) car le système de fichier du système d'exploitation 
est en lecture seule. L'inconvénient est qu'il est figé et donc qu'il 
est impossible d'ajouter de nouvelles applications. L'avantage c'est 
qu'il est comme neuf à chaque redémarrage: les virus et autres malwares 
trépassent. Pour avoir un système d'exploitation avec un système de 
fichier en lecture / écriture il faut utiliser l'option casper-rw du 
mode persistent de l'installeur Live USB, mais vous perdrez les 
possibilités nomades de Cooperation-iws: la clef usb ne sera utilisable 
que sur un seul PC.

* Pour le "pas suffisament d'espace mémoire", cela signifie que toute la 
RAM de votre machine a été utilisée. Lorsque vous installez des 
applications celles ci sont sauvegardées en RAM car votre système de 
fichier est en lecture seule. La solution est d'utiliser une machine 
avec plus de RAM.

* Pour les paramètres wifi, c'est normal car vous avez choisit l'option 
ciws-rw du mode persistent, cela signifie que le répertoire /home de 
votre utilisateur n'est pas sauvegardé. Comme les identifiants de 
connection sont sauvegardés dans le répertoire /home, ils sont perdus à 
chaque redémarrage. Cela sera modifié dans la version 0.9 de 
Cooperation-iws, les répertoires /home et des serveurs seront tous les 
deux persistents.

* Pour le déclenchement du message d'erreur lié à Pyzor, c'est un bug 
lié à l'application Pyzor, cela sera modifié dans la version 0.9. Mais 
c'est un bug non bloquant.

* Pour le déclenchement intempestif du trousseau de clef, cela est 
probablement du à une mauvaise saisie des paramètres de clefs ou au fait 
que le répertoire /home n'est pas persistent. La solution temporaire 
dans ce cas est d'utiliser le serveur avec un cable ethernet (si 
possible). Vous pouvez également saisir les commandes de paramétrages de 
votre environnement wifi dans le fichier /var/share/etc/rc.ciws qui est 
persistent. Ce fichier est lancé automatiquement à chaque démarrage et 
vous permettra de paramétrer automatiquement le wifi.


A bientôt et n'hésitez pas à me poser d'autres questions si mes 
explications ne sont pas claires,

Olivier
